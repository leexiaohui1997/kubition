# 当前 UI 界面描述

> 文档更新时间：2026-03-18（M4 阶段完成后更新）  
> 基于源码版本：`kubition-remake/src`  
> 本文档以自然语言描述当前项目中已实现的所有 UI 界面元素、布局结构、视觉风格和交互行为。

---

## 1. 总体概览

这是一款 **MUD 风格放置 RPG 游戏**（"超苦逼冒险者 - 复刻版"）的前端界面。整体采用 **深色终端风格**，全局等宽字体（JetBrains Mono 优先），视觉上营造出经典文字冒险游戏的氛围。

### 技术栈
- **框架**：React 18 + TypeScript
- **样式**：Tailwind CSS（自定义 MUD 主题色）
- **状态管理**：Zustand
- **滚动方案**：BetterScroll（自定义 Hook 封装）
- **构建工具**：Vite

## 2. 全局视觉风格（MUD 主题）

### 2.1 色彩体系

| 用途 | 色名 | 色值 | 说明 |
|------|------|------|------|
| 主背景 | `mud-bg` | `#0a0a0a` | 近乎纯黑的深色背景 |
| 次级背景 | `mud-bg-secondary` | `#111111` | 面板/卡片的背景色 |
| 三级背景 | `mud-bg-tertiary` | `#1a1a1a` | 边框、分隔线、按钮 hover 态 |
| 主文字 | `mud-text` | `#c8c8c8` | 浅灰色正文 |
| 次级文字 | `mud-text-dim` | `#666666` | 灰暗的提示/辅助文字 |
| 强调色 | `mud-accent` | `#d4a017` | 金黄色，用于标题和重要元素 |
| 成功 | `mud-success` | `#4caf50` | 绿色，成功日志 |
| 警告 | `mud-warning` | `#ff9800` | 橙色，警告日志和低值提示 |
| 信息 | `mud-info` | `#2196f3` | 蓝色，普通操作按钮文字 |
| 危险 | `mud-danger` | `#f44336` | 红色，错误日志 |

### 2.2 字体

全局使用等宽字体栈：`JetBrains Mono` → `Fira Code` → `Cascadia Code` → `Consolas` → `monospace`，营造终端/命令行的视觉效果。

### 2.3 全局样式类（index.css）

定义了三个复用组件类：
- **`.mud-panel`**：深色面板容器（次级背景 + 暗灰边框 + 圆角 + 内边距）
- **`.mud-btn`**：MUD 风格按钮（暗底 + 金色边框/文字，hover 时反色）
- **`.mud-log`**：日志文本区域（纯黑底 + 暗灰边框 + 等宽字体）

### 2.4 动画

- **`log-in`**：日志条目入场动画，从左侧偏移 12px 处淡入滑出，时长 0.2s，ease-out 缓动。
- **`fade-in`**：遮罩层淡入动画，从 `opacity: 0` → `opacity: 1`，时长 0.15s。
- **`slide-up`**：弹层面板入场动画，从 `translateY(16px) scale(0.97)` → `translateY(0) scale(1)`，时长 0.2s。
- **`animate-pulse`**（Tailwind 内置）：用于低值警告文字的闪烁提示和进度条

整个页面是一个 **单页应用（SPA）**，入口组件 `App.tsx` 定义了页面的整体骨架。

│  │                                               │  │

│  │  └───────────────────────┘                    │  │
│  │                                               │  │
│  │  ┌──── 主内容区 ─────────────────────┐       │  │

- **外层**：`min-h-screen`，全屏深黑背景，全局等宽字体，左右内边距 `px-4`（移动端）/ `0`（PC 端）。
- **内容容器**：最大宽度 `max-w-4xl`（约 896px），水平居中，顶部内边距 `py-4`，子元素间距 `space-y-4`。

### 3.2 响应式策略

使用 Tailwind 的 `md:` 断点（768px）做 **移动端优先** 响应式设计：

| 区域 | 移动端（< 768px） | PC 端（≥ 768px） |
|------|-------------------|-------------------|
| 主内容区 | 上下堆叠（flex-col） | 左右分栏（flex-row） |
| 场景区 | 满宽（w-full） | 占 2/3（w-2/3） |
| 日志面板 | 满宽（w-full） | 占 1/3（w-1/3） |
| 日志高度 | h-40（160px） | h-48（192px） |
| 状态项 | 2×2 网格（grid-cols-2） | 水平一行（flex） |
| 游戏时间 | 显示在状态栏顶部 | 显示在状态栏右侧 |
| 标题字号 | text-2xl | text-3xl |
| 副标题字号 | text-xs | text-sm |

### 4.2 状态栏（StatusBar）

**文件**：`src/components/StatusBar.tsx`

深色面板容器（次级背景 + 暗灰边框 + 圆角 + 内边距 `p-3`），展示玩家的四项核心状态和游戏时间。

#### 4.2.1 四项状态条

每项状态包含：

| 状态项 | 关键字 | 进度条颜色 | 低值颜色 |
|--------|--------|------------|----------|
| 生命 | `hp` | 红色（`bg-red-500`） | 暗红（`bg-red-700`） |
| 体力 | `ps` | 黄色（`bg-yellow-500`） | 暗黄（`bg-yellow-700`） |
| 饱食 | `full` | 橙色（`bg-orange-500`） | 暗橙（`bg-orange-700`） |
| 水分 | `moist` | 蓝色（`bg-blue-500`） | 暗蓝（`bg-blue-700`） |

**每项状态的视觉结构**：

```
标签名              数值
体力              65/100
[████████████░░░░░░░░]  ← 进度条（h-2，圆角全满）
```

- **标签**：小字号粗体，正常时为 `text-mud-text`（浅灰），**低于 20% 时变为 `text-mud-warning`（橙色）并闪烁**。
- **数值**：右对齐，暗灰色，格式为 `当前值/最大值`（当前值向下取整）。
- **进度条**：高度 2px（h-2），黑色底槽（`bg-mud-bg`），彩色填充条带有 300ms 宽度过渡动画。**低于 20% 时切换为暗色并闪烁**。

#### 4.2.2 游戏时间

- 格式：`⏰ 第 X 天 HH:00`
- **移动端**：显示在状态栏顶部，暗灰色小字号。
- **PC 端**：显示在状态条右侧，通过左侧竖线（`border-l`）与状态条分隔，`ml-auto` 推到最右。

#### 4.2.3 警告提示区

当玩家状态触发警告时（由 `usePlayerStatus` Hook 判定），在状态栏底部显示警告条目：

- 通过顶部分隔线（`border-t`）与上方内容区分。
- 每条警告：小字号，橙色（`text-mud-warning`），带闪烁动画，前缀 `⚠`。
- 触发条件：
  - **极度饥饿**（饱食 < 20）：`"极度饥饿！生命值恢复速度下降"`
  - **感到饥饿**（饱食 < 50）：`"感到饥饿"`
  - **极度口渴**（水分 < 20）：`"极度口渴！理智值正在下降"`
  - **感到口渴**（水分 < 50）：`"感到口渴"`
  - **体力不足**（体力 < 20）：`"体力不足，需要休息"`

### 4.3 场景路由（SceneRouter）

**文件**：`src/components/scenes/SceneRouter.tsx`

根据 `gameStore.currentPlace` 动态加载对应的场景组件。内部维护一个 `SCENE_MAP` 映射表（`town` → `TownScene`，`forest` → `ForestScene`），未注册的地点自动渲染 `GenericScene`（通用场景组件）。

#### 4.3.1 进度条覆盖层

当玩家正在移动或采集时，场景区上方会覆盖一个半透明遮罩层（`bg-mud-bg/90`），居中显示进度条：

**移动进度条**（`isTraveling` 为 true 时）：
- 提示文字：`"正在前往 {目标地点}..."`，蓝色（`text-mud-info`），带闪烁动画。
- 进度条：宽 3/4，高 3px，蓝色填充（`bg-mud-info`），带 100ms 线性过渡。
- 百分比数字：暗灰色小字号。

**采集进度条**（`isGathering` 为 true 时）：
- 提示文字：`"采集中..."`，绿色（`text-mud-success`），带闪烁动画。
- 进度条：同上结构，绿色填充（`bg-mud-success`）。

**制造进度条**（`isCrafting` 为 true 时）：
- 提示文字：`"制造中..."`，金色（`text-mud-accent`），带闪烁动画。
- 进度条：同上结构，金色填充（`bg-mud-accent`）。

### 4.4 城镇场景（TownScene）

**文件**：`src/components/scenes/TownScene.tsx`

银溪镇场景，基于 `PLACES.town` 数据驱动。深色面板容器（`p-4`）。

#### 4.4.1 场景标题

从地点数据读取名称（`place.name`），金色（`text-mud-accent`），`text-lg` 字号，粗体，底部间距 `mb-2`。

#### 4.4.2 场景描述

一段 MUD 风格的文字描述，暗灰色（`text-mud-text-dim`），小字号（`text-sm`），行距宽松（`leading-relaxed`），支持自动换行（`break-words`）。

> "你站在银溪镇的中央广场上。微风吹过破旧的招牌，远处传来铁匠铺的锤击声。这座小镇虽然不大，却是冒险者们重要的补给站。四周的建筑显得有些老旧，但街道上行人络绎不绝。"

#### 4.4.3 操作按钮列表

标题提示：`"可执行的操作："`，暗灰色极小字号。

每个按钮的通用视觉样式：
- **全宽块级按钮**（`block w-full text-left`）
- **等宽字体**，小字号（`text-sm`）
- **最小触摸高度 44px**（`min-h-[44px]`），确保移动端友好
- **内边距**：`px-3 py-2`，圆角
- **hover 态**：背景变为 `bg-mud-bg-tertiary`
- **禁用态**：半透明 + 禁止光标（移动中或采集中时禁用）

按钮格式遵循经典 MUD 风格 `[序号] 操作名称`：

| 按钮 | 文字颜色 | 行为 |
|------|----------|------|
| `[1] 休息一会儿` | 蓝色（`text-mud-info`） | 恢复 20 体力 + 成功日志 + 推进 2 小时 |
| `[2+] 资源采集` | 蓝色（`text-mud-info`） | 基于 `PLACES.town.resources` 数据动态生成，触发采集进度条 |
| `[N] 前往静谧森林` | 橙色（`text-mud-warning`） | 触发移动进度条，到达后切换到森林场景 |
| `[N] 进入铁匠铺` | 蓝色（`text-mud-info`） | 切换到铁匠铺场景（`SmithScene`） |
| `[N] 进入炼金台` | 蓝色（`text-mud-info`） | 切换到炼金台场景（`AlchemyScene`） |

### 4.6b 物品名称组件（ItemName）

**文件**：`src/components/ItemName.tsx`

通用物品名称展示组件，根据物品 ID 从数据层查找物品信息，并以对应类型颜色渲染名称。

- **颜色**：根据物品类型从 `ITEM_TYPE_COLOR` 常量取色（食材绿、武器红、装备蓝等）。
- **截断**：支持 `truncate` 属性，开启时超长名称以省略号截断（用于背包列表等空间有限场景）。
- **降级**：若物品 ID 不存在于数据层，显示原始 ID 字符串（暗灰色）。
- **使用场景**：背包物品行、配方列表（名称/材料/所需科技）、物品浮窗等所有需要展示物品名称的地方统一使用此组件。

### 4.6c 配方列表组件（RecipeList）

**文件**：`src/components/ui/RecipeList.tsx`

展示指定分类的配方列表，负责数据计算和三态分组排序，渲染委托给 `RecipeItem` 子组件。

#### 三态分组排序

配方按以下优先级排序展示：

1. **可制作**（`craftable`）：材料充足且科技已解锁
2. **材料不足**（`missing`）：科技已解锁但材料不够
3. **未解锁**（`locked`）：所需科技尚未研究

#### 空状态

无配方时显示 `"暂无可用配方"`，暗灰色极小字号。

---

### 4.6d 配方条目组件（RecipeItem）

**文件**：`src/components/ui/RecipeItem.tsx`

单个配方的展示单元，深色面板容器（次级背景 + 暗灰边框 + 圆角 + 内边距 `px-3 py-2`）。

#### 主行结构

```
[🔒] 物品名称（Xh）          [获得材料]  [制作]
```

- **锁定图标**：仅 `locked` 状态显示 🔒 前缀。
- **物品名称**：使用 `ItemName` 组件渲染，带类型颜色。
- **耗时**：`（Xh）`，暗灰色极小字号，紧跟名称后。
- **整行透明度**：`missing` 状态 `opacity-60`，`locked` 状态 `opacity-40`。

#### 调试按钮【获得材料】

仅在 `DEBUG_MODE = true` 时显示，位于主行右侧：
- 样式：橙色边框（`border-amber-600`）+ 橙色文字（`text-amber-500`），hover 时橙色背景填充。
- 点击效果：遍历配方所需材料，逐个调用 `addItem` 加入背包，并写入 `warning` 类型日志：`获得材料：xxx x2、yyy x1`。

#### 制作按钮【制作】

位于主行最右侧：
- **可制作且未禁用**：主题色边框（`border-mud-primary`）+ 主题色文字，hover 时背景填充，`cursor-pointer`。
- **不可制作或操作禁用**：暗灰色边框 + 暗灰文字，`cursor-not-allowed`。
- 点击触发 `onCraft(recipe)` 回调，由 `useCrafting.startCraft` 处理。

#### 材料需求行

```
材料：木材 x3 / 石块 x5
```

- 暗灰色极小字号，`/` 分隔各材料。
- 每个材料名称使用 `ItemName` 组件渲染（带类型颜色）。

#### 缺少材料提示（仅 missing 状态）

```
缺少：木材 x2、石块 x3
```

- 红色（`text-mud-danger`）极小字号，`、` 分隔。
- 材料名称同样使用 `ItemName` 组件渲染。

#### 科技锁定提示（仅 locked 状态）

```
需要科技：制作速度 Lv.1
```

- 橙色（`text-mud-warning`）极小字号。
- 科技名称使用 `ItemName` 组件渲染。

### 4.7 日志面板（LogPanel）

**文件**：`src/components/LogPanel.tsx`

PC 端占主内容区右侧 1/3 宽度，深色面板容器（`p-3`），纵向 flex 布局。

#### 4.4.1 标题栏

水平布局，左右两端对齐：
- **左侧标题**：`"冒险日志"`，金色（`text-mud-accent`），小字号粗体。
- **右侧清除按钮**：`"[清除]"`，暗灰色，hover 时变为橙色（`text-mud-warning`），带颜色过渡动画。

#### 4.4.2 日志列表区

- **容器**：纯黑背景（`bg-mud-bg`），圆角，内边距 `p-2`，等宽字体，极小字号（`text-xs`）。
- **高度**：移动端 `h-40`（160px）/ PC 端 `h-48`（192px），`overflow-hidden`（由 BetterScroll 接管滚动）。
- **滚动方案**：使用 `useBetterScroll` Hook，启用纵向滚动、鼠标滚轮支持、可交互滚动条（淡入淡出式）、DOM 变化自动刷新。新日志添加后自动滚动到顶部（`autoScrollTop: true`）。

#### 4.4.3 日志条目

**空状态**：当无日志时显示 `"暂无日志..."`，暗灰色斜体。

**有日志时**，每条日志的结构：

```
[第1天 08:00] 你休息了一会儿，恢复了些体力
```

- **时间戳**：格式 `[第X天 HH:00]`，暗灰色（`text-mud-text-dim`）。
- **日志正文**：根据日志类型着色——
  - `success`：绿色（`text-mud-success`）
  - `warning`：橙色（`text-mud-warning`）
  - `error`：红色（`text-mud-danger`）
  - `info`：暗灰色（`text-mud-text-dim`）
- **入场动画**：每条日志带有 `animate-log-in` 动画（从左侧淡入滑出，0.2s）。
- **排列顺序**：最新日志在顶部（头部插入）。
- **数量上限**：最多保留 100 条日志，超出后自动截断旧日志。

### 4.8 背包弹层（Modal + Inventory）

背包由两个组件组合实现：通用弹层容器 `Modal` + 背包内容组件 `Inventory`。

#### 4.8.1 通用弹层（Modal）

**文件**：`src/components/Modal.tsx`

所有弹层面板（背包、技能树、商店等）的通用容器。

**遮罩层**：
- `position: fixed`，全屏覆盖，`z-50`。
- 半透明黑色背景（`bg-black/60`）+ 毛玻璃效果（`backdrop-blur-sm`）。
- 点击遮罩关闭弹层。
- 入场动画：`animate-fade-in`（淡入 0.15s）。

**面板**：
- 居中显示（`flex items-center justify-center`）。
- 默认宽度 `w-[90vw] max-w-md`，可通过 `widthClass` 自定义。
- 深色背景（`bg-mud-bg-secondary`）+ 暗灰边框 + 圆角 + 阴影。
- 入场动画：`animate-slide-up`（从下方滑入 + 轻微缩放）。
- 支持 ESC 键关闭。

**标题栏**（可选）：
- 左侧标题：金色（`text-mud-accent`），小字号粗体。
- 右侧区域：可选的 `extra` 插槽（如背包物品计数）+ `[关闭]` 按钮（暗灰色，hover 变橙色）。

#### 4.8.2 背包内容（Inventory）

**文件**：`src/components/Inventory.tsx`

纯内容组件，嵌入 Modal 中使用。通过 `useBetterScroll` 提供列表滚动。

**标题栏额外信息**（`InventoryExtra` 组件）：
- 显示 `物品数/最大容量`，极小字号。
- 接近满载（> 90%）时变红色（`text-mud-danger`），否则暗灰色。

**列表区域**：
- 高度：移动端 `h-60` / PC 端 `h-72`，纯黑背景（`bg-mud-bg`），圆角。
- 空状态：`"背包是空的..."`，暗灰斜体。

**物品行**（每行一个物品）：

```
木材  [食材]  x5                [使用]
```

- **物品名称**：浅灰色（`text-mud-text`），截断溢出。包裹在 `ItemTooltip` 中，hover 时显示详情浮窗。
- **类型标签**：`[食材]`、`[武器]` 等，极小字号，根据物品类型着色（食材绿、武器红、装备蓝等，来自 `ITEM_TYPE_COLOR` 常量）。类型中文名来自 `ITEM_TYPE_LABEL` 常量。
- **数量**：`x{count}`，暗灰色。
- **使用按钮**：仅对可用物品（`item.canUse`）显示，`[使用]` 绿色文字，hover 变浅绿色。
- **行间分隔**：暗灰色底部边框（`border-b`），最后一行无边框。

### 4.9 物品信息浮窗（ItemTooltip）

**文件**：`src/components/ItemTooltip.tsx`

通用物品详情浮窗，hover 物品名称时显示。

#### 4.9.1 定位方案

- 通过 `React.createPortal` 渲染到 `<body>` 层级，彻底脱离 Modal / BetterScroll 的 `transform`/`backdrop-filter` 容器。
- `position: fixed` 基于视口定位，通过 `getBoundingClientRect()` 获取触发元素的精确视口坐标。
- 手动计算位置：默认在触发元素上方，上方空间不足时翻转到下方；水平居中对齐，自动修正边界溢出。
- `z-[9999]`，确保在所有层级之上。

#### 4.9.2 交互方式

- **PC 端**：鼠标 hover 150ms 延迟后显示，移出立即关闭。
- **移动端**：手指按住（touchstart）显示，松开（touchend）关闭。
- **入场动画**：150ms 淡入 + 从下方 4px 滑入（CSS transition）。
- **鼠标穿透**：浮窗设置 `pointerEvents: none`，不影响下方元素交互。

#### 4.9.3 浮窗内容

深色面板（`bg-mud-bg-secondary`），暗灰边框，圆角，最大宽度 `max-w-64`，等宽字体极小字号。

**首行：名称 + 类型（同行显示）**

```
石斧  [武器]
```

- **物品名称**：`text-sm` 粗体，浅灰色（`text-mud-text`）。
- **类型标签**：`[食材]`、`[武器]` 等，`text-[10px]` 粗体，根据物品类型着色（来自 `ITEM_TYPE_COLOR`），中文名来自 `ITEM_TYPE_LABEL`。
- 两者通过 `flex items-baseline gap-1.5` 排列在同一行，类型标签基线对齐名称底部。

根据物品类型动态展示不同属性区块：

| 区块 | 条件 | 内容 |
|------|------|------|
| 名称 + 类型 | 有 `item.name` 或 `item.type` | 同行展示，名称粗体大字，类型彩色小标签 |
| 描述 | 有 `item.desc` | 物品描述文字 |
| 价值 | `item.value > 0` | `"价值: X 金"`，金色标签 |
| 食物/药剂效果 | 有 `item.effect` | 各属性增减值（正值绿色，负值红色），中文标签来自 `EFFECT_LABEL` |
| 武器属性 | 有 `item.weaponType` | 武器类型（来自 `WEAPON_TYPE_LABEL`）、伤害、射程、耐久、特殊效果（冻结/恐惧/格挡/诅咒/吸血） |
| 装备属性 | 有 `item.equipType` | 装备槽位（来自 `EQUIP_SLOT_LABEL`）、伤害加成、减伤、采集/移动速度、温度相关等 |

## 5. 交互行为汇总

| 交互 | 触发方式 | 效果 |
|------|----------|------|
| 点击操作按钮 | 点击场景中的操作按钮 | 添加日志 + 推进游戏时间 + 触发状态消耗/恢复 |
| 采集资源 | 点击场景中的采集按钮 | 显示采集进度条覆盖层 → 完成后获得物品 + 日志反馈 |
| 移动到其他地点 | 点击场景中的移动按钮 | 显示移动进度条覆盖层 → 完成后切换到目标场景 |
| 打开/关闭背包 | 点击快捷操作栏 `[I] 背包` | 打开/关闭背包弹层（Modal） |
| 关闭弹层 | 点击遮罩 / 按 ESC / 点击 `[关闭]` | 关闭当前弹层 |
| 查看物品详情 | hover 背包中物品名称 / 移动端按住 | 显示物品信息浮窗（ItemTooltip） |
| 使用物品 | 点击背包物品行的 `[使用]` 按钮 | 使用物品（恢复属性等）+ 日志反馈 |
| 清除日志 | 点击日志面板 `[清除]` 按钮 | 清空所有日志条目 |
| 日志滚动 | 鼠标滚轮 / 拖拽滚动条 | 在日志面板内滚动浏览历史日志 |
| 新日志自动滚顶 | 日志数量变化时自动触发 | 日志面板平滑滚动到顶部（300ms 动画） |
| 状态低值闪烁 | 状态值低于 20% 时自动触发 | 对应的标签和进度条变暗色并闪烁 |
| 移动/采集中禁用 | 移动或采集进行中时自动触发 | 所有操作按钮变半透明且不可点击 |

## 7. 常量管理

项目将 UI 相关的常量集中管理在 `src/constants/` 目录下，避免各组件重复定义：

| 文件 | 职责 | 典型常量 |
|------|------|----------|
| `constants/labels.ts` | 中文标签映射 | `EFFECT_LABEL`（效果属性）、`WEAPON_TYPE_LABEL`（武器类型）、`EQUIP_SLOT_LABEL`（装备槽位）、`ITEM_TYPE_LABEL`（物品类型） |
| `constants/styles.ts` | UI 样式映射 | `ITEM_TYPE_COLOR`（物品类型颜色）、`LOG_COLOR`（日志颜色）、`STAT_CONFIGS`（状态栏配置） |
| `constants/game.ts` | 游戏数值 | 状态上限/消耗速率、进度条参数（`GATHER_TICK_INTERVAL`、`TRAVEL_TICK_INTERVAL` 等）、`DEBUG_MODE`（调试模式开关） |
